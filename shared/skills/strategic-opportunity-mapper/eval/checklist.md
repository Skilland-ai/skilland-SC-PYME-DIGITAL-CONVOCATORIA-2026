# Eval checklist — strategic-opportunity-mapper

## Structure

- [ ] Existe strategic opportunity brief.
- [ ] Existe handoff to QA Gate 1.
- [ ] Ambos viven dentro de `02_strategy/`.
- [ ] El handoff usa status válido del workflow.

## Intake handling

- [ ] Lee intake brief.
- [ ] Lee handoff to strategy.
- [ ] No ignora unknowns del intake.
- [ ] No inventa información no soportada.

## Strategic quality

- [ ] Formula una tesis estratégica clara.
- [ ] Identifica actores, incentivos y roles.
- [ ] Define qué juego se está jugando.
- [ ] Identifica ángulos de oportunidad.
- [ ] Identifica riesgos estratégicos.
- [ ] Identifica no-go zones.
- [ ] Define criterios de éxito.

## Reasoning hygiene

- [ ] Separa facts, hypotheses, assumptions y unknowns.
- [ ] Marca decisiones que requieren usuario.
- [ ] No oculta supuestos sensibles.
- [ ] No convierte hipótesis en hechos.

## Scope control

- [ ] No diseña oferta.
- [ ] No crea packs.
- [ ] No define pricing.
- [ ] No define ICP.
- [ ] No diseña funnel.
- [ ] No crea artefactos comerciales.
- [ ] No ejecuta QA Gate 1.

## Handoff quality

- [ ] El QA Gate 1 puede usar el handoff.
- [ ] Indica qué QA debe revisar cuidadosamente.
- [ ] Indica qué downstream no debe asumir todavía.
- [ ] Recomienda una acción siguiente clara.
- [ ] Actualiza o instruye actualización de run state.

## Harness integration

- [ ] Lee `00_run_state.md` antes de escribir.
- [ ] Verifica que intake está listo o bloquea con el status correcto.
- [ ] Escribe solo dentro de `02_strategy/`.
